## 方案概述

### 背景

原计划使用 Server酱（微信推送），但仅支持单向推送，无法接收消息。
改用飞书 WebSocket 长连接，实现「接收消息 → 分析 → 推送报告」的闭环。

### 改造原则

1. **最小化改动**：优先改造现有代码，不新增过多模块
2. **先基础后增强**：先实现请求-响应模式，暂缓流式推送
3. **复用现有代码**：FeishuNotifier、NotificationHub 等可复用

### 技术选型

| 组件 | 方案 | 说明 |
|------|------|------|
| 消息接收 | 飞书 WebSocket 长连接 | 本地主动连接飞书服务器，无需公网 |
| 消息推送 | 飞书 Webhook | 现有 FeishuNotifier 复用 |
| SDK | lark-oapi | 官方 Python SDK，支持 WebSocket |

## 数据流

### 消息流程

```
用户发送: "000001"
    ↓
飞书服务器 (WebSocket)
    ↓
listener.py (接收消息)
    ↓
handler.py (解析股票代码)
    ↓
main.py (执行分析流程)
    ↓
feishu.py (推送报告到用户)
    ↓
用户收到: 分析报告
```

## 代码改造清单

### 1. src/notification/feishu.py（改造）

**改造内容**：新增 `send_to_user()` 方法，支持向指定用户推送消息

```
┌─────────────────────────────────────────┐
│         FeishuNotifier 改造              │
├─────────────────────────────────────────┤
│ + send()           # 现有 Webhook 群发  │
│ + send_to_user()   # 新增：用户定向推送 │
```

**新增方法签名**：
```python
def send_to_user(self, open_id: str, content: str, title: str = None) -> bool:
    """向指定用户发送消息

    Args:
        open_id: 飞书用户 OpenID
        content: 消息内容
        title: 可选的标题

    Returns:
        是否发送成功
    """
```

### 2. src/main.py（改造）

**改造内容**：新增 `--feishu-ws` 参数，启动 WebSocket 监听模式

```
启动流程：
    ↓
解析 --feishu-ws 参数
    ↓
初始化 FeishuWebSocketListener
    ↓
启动长连接
    ↓
监听消息 → 处理 → 推送报告
```

### 3. src/config/__init__.py（改造）

**新增配置项**：

| 配置项 | 说明 |
|--------|------|
| `FEISHU_APP_ID` | 飞书应用 App ID |
| `FEISHU_APP_SECRET` | 飞书应用 App Secret |
| `FEISHU_WS_ENABLED` | 是否启用 WebSocket 模式 |

## 风险与注意事项

### 1. 消息限制

- 飞书机器人每秒最多 20 条消息
- 单条消息最大 30KB
- 超长报告需要分片发送

### 2. 长连接稳定性

- SDK 会自动重连
- 建议增加心跳机制

### 3. 命令冷却

- 建议同一用户 30 秒内只处理一次分析请求
- 避免重复分析消耗资源
