## 项目概览

### 项目是什么？

这是一个**多 Agent 协作的 A 股分析系统**。简单说：

```
你输入股票代码 → 6 个 AI Agent 并行分析 → 综合报告回给你
```

6 个 Agent 分别是：量化分析师、图表分析师、情报官、风险评估师、基本面分析师、舆情监控师。

它们同时工作，各自给出分析结论，最后由"首席投资官"（Supervisor）汇总成一份完整报告。

### 技术栈

| 技术 | 用途 |
|------|------|
| **LangGraph** | 工作流编排（让 6 个 Agent 并行工作） |
| **MiniMax API** | 大模型（生成分析结论） |
| **AKShare** | A 股数据源（K线、资金流、财务数据） |
| **DuckDuckGo** | 联网搜索（新闻、舆情） |

## 核心概念解析

### 1. 什么是 Agent？

在 LangGraph 里，**Agent 就是一个函数**：接收 `AgentState`（状态），返回 `AgentState`（状态）。

```python
def agent_node(state: AgentState) -> AgentState:
    # 做一些分析...
    state.some_result = result
    return state
```

每个 Agent 负责一个专业领域的分析，然后把结果写回状态里。

### 2. 什么是 AgentState？

`AgentState` 是整个工作流使用的**共享黑板**，所有 Agent 都往上面写自己的结果：

```python
class AgentState(BaseModel):
    query: StockQuery                      # 用户的问题（股票代码是什么）
    quantitative_result: AgentResult       # 量化分析师的结果
    chart_result: AgentResult             # 图表分析师的结果
    # ... 6 个 agent 的结果都放这里
    final_report: str                      # 最终报告（Supervisor 生成）
    completed_tasks: List[str]            # 记录哪些 Agent 完成了
```

### 3. 什么是 Supervisor？

Supervisor 是整个系统的**协调者**，类似公司里的 CEO：

- **第一次执行**：检查哪些 Agent 结果还没回来，设置"路由目标"让工作流去并行执行
- **第二次执行**：所有 Agent 结果都回来了，汇总成最终报告

### 4. 什么是 Send API？

LangGraph 的 `Send` API 用于**并行分发任务**：

```python
def parallel_dispatch(state):
    return [
        Send("quantitative", state),   # 把任务发给量化分析师
        Send("chart", state),           # 把任务发给图表分析师
        # 6 个 Send 同时发出，6 个 Agent 同时运行
    ]
```

这就是为什么 6 个 Agent 能"真并行"——每个 Agent 是独立启动的。

## LangGraph 工作流详解

### 工作流图（ASCII 版）

```
                    ┌─────────────┐
                    │  Supervisor │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │                         │
              ▼                         ▼
    routing_target="parallel"     final_report 存在
              │                         │
              ▼                         ▼
    ┌─────────────────────────────────┐
    │      parallel_phase             │
    │      (Send API 并行分发)         │
    │                                 │
    │  ┌─────┐ ┌─────┐ ┌─────┐       │
    │  │ Q   │ │ C   │ │ I   │       │
    │  └─────┘ └─────┘ └─────┘       │
    │  ┌─────┐ ┌─────┐ ┌─────┐       │
    │  │ R   │ │ F   │ │ S   │       │
    │  └─────┘ └─────┘ └─────┘       │
    └─────────────────────────────────┘
              │
              ▼
    （各 Agent 完成后返回 Supervisor）
              │
              ▼
    ┌─────────────────┐
    │ Supervisor 再跑 │
    │ 生成 final_report│
    └─────────────────┘
              │
              ▼
            END
```

### 条件边逻辑（核心）

```python
def supervisor_routing(state):
    # 1. 如果已经有报告了，直接 END
    if state.final_report:
        return END

    # 2. 检查还缺哪些 Agent 结果
    missing = ["quantitative_result", "chart_result", ...]
    has_missing = any(getattr(state, r) is None for r in missing)

    # 3. 如果还有缺失，路由到并行阶段
    if has_missing and state.routing_target == "parallel":
        return parallel_dispatch(state)  # Send API 分发任务

    # 4. 如果都齐了，让 Supervisor 再跑一次（生成报告）
    if not has_missing:
        return "supervisor"

    # 5. 默认 END
    return END
```

**为什么 Supervisor 会被触发两次？**

- 第一次：结果还没齐，设置 `routing_target="parallel"`，触发并行执行
- 中间：6 个 Agent 并行运行，完成后各自返回 Supervisor
- 第二次：所有结果都齐了，Supervisor 生成最终报告

## Agent 架构与职责

### 6 个 Agent 一览

| Agent | 权重 | 输入数据 | 输出 |
|-------|------|----------|------|
| 量化分析师 | 25% | K线 + 资金流向 | 技术面评分、趋势判断 |
| 图表分析师 | 15% | K线形态 | 形态评分、支撑阻力位 |
| 情报官 | 10% | 搜索新闻 | 政策/行业动态评分 |
| 风险评估师 | 20% | K线 + 波动率 | 风险评分、**止损位** |
| 基本面分析师 | 20% | 财务数据 | 估值评分、盈利能力 |
| 舆情监控师 | 10% | 搜索舆情 | 情绪评分、分析师评级 |

### 每个 Agent 的分析流程（都是 4 步）

```
1. 获取数据（AKShare / DuckDuckGo 搜索）
2. 计算指标（技术指标 / 形态识别 / ...）
3. 调用 LLM（MiniMax 生成分析结论）
4. 写回状态（AgentResult 写入 state.xxx_result）
```

### AgentResult 是什么？

每个 Agent 分析完成后，会生成一个 `AgentResult` 放到状态里：

```python
class AgentResult(BaseModel):
    agent_name: str           # "quantitative" / "chart" / ...
    score: float              # 0-100，技术面评分
    confidence: float          # 0-1，数据质量置信度
    conclusion: str           # 分析结论（一句话）
    recommendation: str      # 建议（非投资建议）
    key_findings: List[str]   # 3-5 个关键发现
    raw_data: Dict            # 原始数据（如止损位）
    data_quality: DataQuality # 数据质量指标
```

## 关键文件解读

### 1. main.py - 入口文件

```python
# 简化流程
def main():
    # 1. 解析命令行参数
    args = parser.parse_args()

    # 2. 构建工作流
    graph = build_stock_analysis_graph(weight_mode=args.weight_mode)

    # 3. 构建初始状态
    state = AgentState(query=StockQuery(ts_code="000001.SZ", ...))

    # 4. 流式执行工作流（打印每个 Agent 的进度）
    for event in graph.stream(state):
        print(event)  # 每个事件包含 node_name 和状态

    # 5. 发送通知
    hub.send(content=final_report, title="分析报告", channels=["feishu"])
```

### 2. builder.py - 工作流构建

```python
def build_stock_analysis_graph():
    # 1. 创建 7 个节点（6 个 Agent + Supervisor）
    workflow.add_node("supervisor", supervisor_fn)
    workflow.add_node("quantitative", quantitative_fn)
    # ... 其他 5 个 Agent

    # 2. 设置入口点
    workflow.set_entry_point("supervisor")

    # 3. 添加条件边（路由逻辑）
    workflow.add_conditional_edges(
        "supervisor",
        supervisor_routing,  # 这个函数决定下一步去哪
        ["supervisor", "quantitative", "chart", ..., END]
    )

    # 4. 各 Agent 完成后返回 Supervisor
    workflow.add_edge("quantitative", "supervisor")
    workflow.add_edge("chart", "supervisor")
    # ...

    # 5. 编译工作流
    return workflow.compile()
```

### 3. state.py - 状态定义

```python
class AgentState(BaseModel):
    # 输入
    query: StockQuery                    # 股票代码 + 日期范围

    # 6 个 Agent 的结果（并行执行时各自填充）
    quantitative_result: AgentResult = None
    chart_result: AgentResult = None
    intelligence_result: AgentResult = None
    risk_result: AgentResult = None
    fundamental_result: AgentResult = None
    sentiment_result: AgentResult = None

    # 输出
    final_report: str = None             # Supervisor 生成的报告

    # 路由控制
    routing_target: str = None           # "parallel"=分发任务, None=结束
    completed_tasks: List[str] = []     # 已完成的 Agent 列表
```

### 4. supervisor.py - 首席投资官

Supervisor 做了两件事：

**第一次执行（routing_target=None）：**
```python
# 检查还有哪些结果没回来
missing = [r for r in required_results if getattr(state, r) is None]
if missing:
    state.routing_target = "parallel"  # 设置路由目标，分发任务
    return state
```

**第二次执行（所有结果都齐了）：**
```python
# 汇总 6 个 Agent 的结果
final_report = _build_fixed_report(state, weights, ...)  # 或 _build_auto_report
state.final_report = final_report
return state
```

## 从输入到输出的完整流程

以 `python -m src.main --stock 000001.SZ --name 平安银行 --notify console` 为例：

### Step 1: CLI 解析
main.py 解析 `--stock` 和 `--name` 参数，构建 `StockQuery(ts_code="000001.SZ", stock_name="平安银行")`

### Step 2: 构建工作流
`build_stock_analysis_graph()` 创建包含 7 个节点的 StateGraph

### Step 3: Supervisor 首次执行
- `supervisor_routing()` 检查 6 个结果都是 None
- `has_missing=True`，`routing_target=None`，不满足 `"parallel"` 条件
- 等等...等等，让我重新看代码逻辑...

实际上 Supervisor 首次执行时：
1. 检查 missing results → 有缺失
2. 设置 `state.routing_target = "parallel"`
3. Supervisor 再次被调用（因为状态变了）

然后条件边 `supervisor_routing` 检查：
- `has_missing=True` 且 `routing="parallel"` → 返回 `parallel_dispatch`
- 并行分发 6 个 Agent

### Step 4: 6 个 Agent 并行执行

每个 Agent 独立运行，互不等待：
- 量化分析师：获取 K线 → 计算 MA/RSI/MACD → 调用 LLM → 写回 `state.quantitative_result`
- 图表分析师：获取 K线 → 识别形态 → 调用 LLM → 写回 `state.chart_result`
- ...（其他 Agent 类似）

### Step 5: 各 Agent 完成后返回 Supervisor

每个 Agent 完成后自动回到 Supervisor：
- Supervisor 再次执行 `supervisor_routing`
- 每次回来检查一次：还有缺失吗？
- 6 个 Agent 全部完成后，`has_missing=False`

### Step 6: Supervisor 生成报告

`supervisor_routing` 返回 `"supervisor"`，Supervisor 节点再次执行：
- 汇总 6 个结果
- 按权重计算综合评分
- 调用 LLM 生成综合报告
- 过 DisclaimerFilter（过滤高风险词汇）
- 注入 MandatoryDisclaimer（免责文本）

### Step 7: 推送通知

`NotificationHub.send()` 把报告发送到飞书/Server酱/钉钉/控制台

## 常见问题 FAQ

### Q: 为什么 6 个 Agent 能并行运行？

因为 LangGraph 的 `Send` API 会同时触发 6 个节点，每个节点独立执行，不需要等待。

### Q: Supervisor 被触发了几次？

理论上 3 次：
1. **首次**：设置 `routing_target="parallel"`，触发并行分发
2. **中间**：每个 Agent 完成后都会回到 Supervisor（6 次）
3. **末次**：所有结果齐了，生成最终报告

### Q: 如何增加新的 Agent？

1. 在 `agents/` 目录下创建新文件，如 `new_agent.py`
2. 定义 `create_new_agent(model)` 函数，返回 `{"analyze": analyze_fn}`
3. 在 `builder.py` 中添加节点和边
4. 在 `state.py` 中添加 `new_agent_result: AgentResult` 字段

### Q: 如何改变 Agent 的权重？

**Fixed 模式**：修改 `config.AGENT_WEIGHTS` 字典
**Auto 模式**：Supervisor 会根据数据质量自动调整

### Q: 数据从哪来？

- **AKShare**：K线、资金流、财务数据（免费开源）
- **DuckDuckGo 搜索**：新闻、舆情、分析师评级（无需 API key）
- **北向资金**：通过 AKShare 获取

### Q: 如何调试？

查看 `src/test_workflow.py`、`src/test_agents.py`、`src/test_debug.py`
