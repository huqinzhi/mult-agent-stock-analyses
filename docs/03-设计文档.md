## 核心业务流程

### 流程概览

```
用户执行命令
    ↓
CLI 解析参数 (main.py)
    ↓
构建 AgentState + StockQuery
    ↓
build_stock_analysis_graph() 构建工作流
    ↓
graph.stream() 流式执行
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Supervisor 首次执行                                          │
│ - 检查 missing results                                       │
│ - 设置 routing_target="parallel"                             │
└─────────────────────────────────────────────────────────────┘
    ↓
条件边路由 → parallel_phase (Send API 并行)
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 6 个 Agent 并行执行                                          │
├─────────────────────────────────────────────────────────────┤
│ 量化分析师 → 技术指标，资金流向，趋势判断                     │
│ 图表分析师 → K线形态，支撑阻力位                             │
│ 情报官 → 宏观政策，行业动态，事件                            │
│ 风险评估师 → 风险量化，止损位计算                            │
│ 基本面分析师 → 财报解读，估值分析                            │
│ 舆情监控师 → 情绪分析，分析师评级                           │
└─────────────────────────────────────────────────────────────┘
    ↓
各 Agent 完成后返回 Supervisor
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Supervisor 二次执行                                          │
│ - 汇总 6 个 Agent 结果                                       │
│ - 计算综合评分（固定权重 or 自动权重）                        │
│ - 生成综合报告（含投资建议章节）                              │
└─────────────────────────────────────────────────────────────┘
    ↓
DisclaimerFilter.filter() 过滤高风险词汇
    ↓
MandatoryDisclaimer.inject() 注入免责文本
    ↓
最终报告
    ↓
NotificationHub.send() 多渠道推送
    ↓
完成
```

**工作流构建** (`builder.py:36-161`):
1. 创建 6 个 Agent + 1 个 Supervisor
2. 添加节点到 StateGraph
3. 设置条件边路由
4. 编译工作流

### Step 4: 各 Agent 分析逻辑

详见 [Agent 详细说明](#agent-详细说明)

### Step 5: Supervisor 汇总

**固定权重模式** (`supervisor.py:150-266`):
1. 按权重汇总: `final_score = Σ(agent_score × weight)`
2. 调用 LLM 生成综合报告
3. 包含投资建议章节

**自动权重模式** (`supervisor.py:269-367`):
1. 根据 data_quality 和 confidence 动态分配权重
2. 输出权重分配理由
3. 计算综合评分
4. 生成综合报告

### Step 6: 投资建议章节

**格式**:
```
## 投资建议
综合评分: XX/100

【操作建议】
- 综合评分 >= 70: 建议买入
- 综合评分 50-69: 建议观望
- 综合评分 < 50: 暂不推荐

【如建议买入】
- 买入区间: XX-XX 元
- 止盈区间: XX-XX 元
- 止损位: XX 元

【如不建议买入】
- 当前风险提示: [主要风险因素]
- 建议关注: [需要关注的风险点]
```

### Step 7: 合规过滤

详见 [合规机制](#合规机制)

### Step 8: 通知推送

**NotificationHub** (`notification/notifier.py`):
- console: 直接输出到终端
- serverchan: 微信推送（Server酱）
- feishu: 飞书群机器人

## 任务分发机制

### 条件边逻辑

**文件**: `builder.py:107-114`

```python
def supervisor_routing(state: AgentState):
    # 已有 final_report → END
    if final_report:
        return END

    # 有缺失结果 + routing="parallel" → 并行分发
    if has_missing and routing == "parallel":
        return parallel_dispatch(state)

    # 无缺失结果 → 继续执行（生成报告）
    if not has_missing:
        return "supervisor"

    return END
```

### 并行分发

**文件**: `builder.py:88-104`

```python
def parallel_dispatch(state: AgentState) -> list:
    return [
        Send("quantitative", state),
        Send("chart", state),
        Send("intelligence", state),
        Send("risk", state),
        Send("fundamental", state),
        Send("sentiment", state),
    ]
```

### LangGraph Send API

使用 LangGraph 的 `Send` API 实现真正的并发执行：
- 6 个 Agent 同时启动
- 各自独立执行分析
- 完成后返回 Supervisor
- Supervisor 汇总结果生成报告

## Agent 详细说明

### 量化分析师 (`src/agents/quantitative_analyst.py`)

**权重**: 25%

**职责**:
1. 调用 AKShare 获取 K线数据
2. 计算技术指标（MA、RSI、MACD、布林带）
3. 获取资金流向数据
4. LLM 分析生成结论

**输出**:
- score (0-100)
- conclusion
- key_findings

**系统提示词**:
```
你是一位量化分析专家，负责技术指标计算和市场数据分析。

你的职责：
1. 计算技术指标（MA、MACD、RSI、布林带等）
2. 分析资金流向（主力净流入、超大单、大单、中单、小单）
3. 趋势判断（上升趋势、下降趋势、横盘震荡）
4. 异常检测（成交量异常、价格异常）
5. 北向资金分析（如有数据）
```

### 图表分析师 (`src/agents/chart_analyst.py`)

**权重**: 15%

**职责**:
1. 调用 AKShare 获取 K线数据
2. 识别 K线形态（锤子线、吞没、十字星等）
3. 分析支撑阻力位
4. LLM 分析生成结论

**输出**:
- score (0-100)
- conclusion
- key_findings

### 情报官 (`src/agents/intelligence_officer.py`)

**权重**: 10%

**职责**:
1. 使用 DuckDuckGo 搜索宏观新闻
2. 搜索行业动态和政策
3. LLM 分析生成结论

**输出**:
- score (0-100)
- conclusion
- key_findings

### 风险评估师 (`src/agents/risk_analyst.py`)

**权重**: 20%

**职责**:
1. 计算风险指标（波动率、VaR、最大回撤）
2. **计算止损位**（波动率止损、支撑位止损、ATR止损）
3. LLM 分析生成结论

**止损位计算** (`risk_analyst.py:98-138`):
```python
def _estimate_stop_loss(df: pd.DataFrame, volatility: float = None) -> Dict[str, Any]:
    """
    估算止损位
    - 波动率止损：2倍年化波动率
    - 20日支撑位止损：近20日最低点
    - ATR止损：2倍 ATR
    """
```

**输出数据结构** (在 `risk_result.raw_data["stop_loss"]`):
```python
{
    "current_price": 10.50,           # 当前价格
    "volatility_stop_pct": 8.5,        # 止损百分比
    "volatility_stop_price": 9.61,     # 波动率止损价格
    "support_20d_price": 9.80,        # 20日支撑位
    "support_20d_pct": 6.67,          # 距支撑位百分比
    "atr_stop_price": 9.95,            # ATR止损位
}
```

### 基本面分析师 (`src/agents/fundamental_analyst.py`)

**权重**: 20%

**职责**:
1. 调用 AKShare 获取财务数据
2. 计算估值指标（PE、PB、PS）
3. 分析盈利能力（ROE、ROA、毛利率）
4. LLM 分析生成结论

**输出**:
- score (0-100)
- conclusion
- key_findings

### 舆情监控师 (`src/agents/sentiment_analyst.py`)

**权重**: 10%

**职责**:
1. 使用 DuckDuckGo 搜索社交媒体情绪
2. 搜索分析师评级
3. LLM 分析生成结论

**输出**:
- score (0-100)
- conclusion
- key_findings

### Supervisor (`src/agents/supervisor.py`)

**职责**:
1. 协调 6 个 Agent 执行
2. 汇总分析结果
3. 计算综合评分
4. 生成综合报告（含投资建议）

**权重模式**:
- Fixed: 按固定权重汇总
- Auto: 根据数据质量和置信度动态分配权重

## 合规机制

### DisclaimerFilter (`src/compliance/disclaimer_filter.py`)

**功能**: 过滤高风险煽动性词汇，保留专业投资建议术语

**过滤规则**:
- 过滤高风险煽动性词汇:
  - "强烈买入" → "建议谨慎买入"
  - "满仓杀入" → "建议控制仓位"
  - "无脑买入" → "建议理性分析"
- **保留专业术语**:
  - "建议买入" - 保留
  - "止损位" - 保留
  - "止盈区间" - 保留

### MandatoryDisclaimer (`src/compliance/mandatory_disclaimer.py`)

**功能**: 在报告尾部注入完整法律免责文本

## LangGraph 状态定义

**文件**: `src/graph/state.py`

### StockQuery
```python
class StockQuery(BaseModel):
    ts_code: str                           # 股票代码，如 000001.SZ
    stock_name: Optional[str] = None        # 股票名称
    query_type: Literal["quick", "comprehensive"] = "comprehensive"
    start_date: Optional[str] = None       # 开始日期
    end_date: Optional[str] = None         # 结束日期
```

### DataQuality
```python
class DataQuality(BaseModel):
    quality_score: float = 0.0            # 0-1，数据质量总分
    completeness: float = 0.0              # 0-1，完整性
    timeliness: float = 0.0                 # 0-1，时效性
    consistency: float = 0.0                # 0-1，一致性
    details: Dict[str, Any] = Field(default_factory=dict)
```

### AgentResult
```python
class AgentResult(BaseModel):
    agent_name: str
    score: Optional[float] = None           # 0-100，评分
    confidence: Optional[float] = None       # 0-1，置信度
    data_quality: Optional[DataQuality] = None
    conclusion: str = ""                    # 分析结论
    recommendation: str = ""               # 建议
    key_findings: List[str] = Field(default_factory=list)
    raw_data: Dict[str, Any] = Field(default_factory=dict)  # 包含 stop_loss 等
```

### AgentState
```python
class AgentState(BaseModel):
    messages: List[Any] = Field(default_factory=list)
    query: Optional[StockQuery] = None

    # 6 个 Agent 结果
    quantitative_result: Optional[AgentResult] = None
    chart_result: Optional[AgentResult] = None
    intelligence_result: Optional[AgentResult] = None
    risk_result: Optional[AgentResult] = None
    fundamental_result: Optional[AgentResult] = None
    sentiment_result: Optional[AgentResult] = None

    # 最终报告
    final_report: Optional[str] = None
    routing_target: Optional[str] = None     # "parallel" 表示分发任务
    completed_tasks: List[str] = Field(default_factory=list)
```
